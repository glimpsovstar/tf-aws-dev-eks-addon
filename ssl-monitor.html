Equivalent Vault CLI Commands

Here's how you would perform the same operations manually using Vault CLI:

üí° Issue Certificate: Vault generates both the private key and certificate
vault write pki-demo/issue/kubernetes \ common_name="nginx-demo.david-joo.sbx.hashidemos.io" \ alt_names="nginx-demo.demo.svc.cluster.local" \ ttl="1h" \ format="pem"
üîß With additional options:
vault write pki-demo/issue/kubernetes \ common_name="nginx-demo.david-joo.sbx.hashidemos.io" \ alt_names="nginx-demo.demo.svc.cluster.local,localhost" \ ip_sans="127.0.0.1,10.1.0.0/16" \ ttl="1h" \ format="pem" \ private_key_format="pkcs8"
üìù Sign CSR: You provide the CSR, Vault signs it
# First, generate a CSR (using openssl) openssl req -new -key private.key -out server.csr \ -subj "/CN=nginx-demo.david-joo.sbx.hashidemos.io"
# Then sign it with Vault vault write pki-demo/sign/kubernetes \ csr=@server.csr \ common_name="nginx-demo.david-joo.sbx.hashidemos.io" \ alt_names="nginx-demo.demo.svc.cluster.local" \ ttl="1h"
üö´ Revoke Certificate: Invalidate a certificate before expiry
# Revoke by serial number vault write pki-demo/revoke \ serial_number=""
# Revoke by certificate file vault write pki-demo/revoke \ certificate=@certificate.pem
üìã List & Information: View certificates and CA details
# List all certificates vault list pki-demo/certs
# Get CA certificate vault read pki-demo/cert/ca
# Get certificate by serial number vault read pki-demo/cert/
# Check role configuration vault read pki-demo/roles/kubernetes
# View CRL (Certificate Revocation List) vault read pki-demo/cert/crl
